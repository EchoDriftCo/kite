## Overview

Recipe form for creating new recipes or editing existing ones. Clean, organized, mobile-first. Dynamic arrays for ingredients and instructions with reorder controls. Import shortcuts at top for new recipes.

**Routes:**
- `/recipes/new` — Create new recipe
- `/recipes/:id/edit` — Edit existing recipe

**Features:**
- Basic info section (title, description, servings, times, source, image URL, visibility toggle)
- Import shortcuts (Import Recipe, Multi-Image Import) — new recipes only
- Ingredients array (qty, unit, item, preparation) with add/remove/reorder
- Instructions array (step text) with add/remove/reorder
- Tags selector (edit mode only)
- Form validation
- Save/Cancel actions
- Unsaved changes warning

## States

### Loading (Initial Load — Edit Mode)
- **Show:** Spinner + "Loading recipe..."
- **Hide:** Form until recipe data loaded

### Saving
- **Disable:** All form fields
- **Disable:** All buttons except Cancel
- **Save button:** Show spinner, text "Saving..."
- **Prevent:** Navigation away (unsaved changes)

### Error (Save Failed)
- **Show:** Error message at top of form
- **Icon:** `error` (24px, `$error`)
- **Text:** Error details or "Failed to save recipe. Please try again."
- **Background:** `rgba(239, 68, 68, 0.1)` (error tint)
- **Border:** 1px solid `$error`
- **Border-radius:** 8px
- **Padding:** 12px
- **Margin-bottom:** 16px

### Success (Saved)
- **Action:** Navigate to `/recipes/:id` (recipe detail)
- **Snackbar:** "Recipe saved successfully!" (success variant)

### Validation Errors
- **Show:** Error messages below invalid fields
- **Trigger:** On field blur (touched) and form submit
- **Error text:** 400 12px, `$error`
- **Icon:** `error` icon (16px) in field

### Unsaved Changes Warning
- **Trigger:** User tries to navigate away with unsaved changes
- **Component:** Dialog (confirmation)
- **Title:** "Unsaved Changes"

## Accessibility

- **Page title:** "Create Recipe — RecipeVault" or "Edit Recipe — RecipeVault"
- **Form landmark:** `<form role="form">`
- **Field labels:** All fields have visible labels (mat-label)
- **Required indicators:** Asterisk (*) or "required" in label
- **Error announcements:** aria-live="assertive" on error messages
- **Ingredient/instruction rows:** 
  - Announce row number via screen reader
  - Reorder buttons: Clear aria-labels ("Move ingredient up", "Move instruction down")
  - Delete buttons: Clear aria-labels ("Remove ingredient 2", "Remove instruction 3")
- **Add buttons:** aria-labels ("Add ingredient", "Add instruction")
- **Focus order:** 
  1. Basic info fields (top to bottom)
  2. Ingredients (first to last, left to right within row)
  3. Instructions (first to last)
  4. Tags
  5. Actions (Cancel, Save)

---

## Validation Rules

**Required Fields:**
- Title
- Servings (min 1)
- At least one ingredient (item field required per ingredient)
- At least one instruction (instruction field required per instruction)

**Optional Fields:**
- Description
- Prep Time
- Cook Time
- Source
- Image URL
- Ingredient: Quantity, Unit, Preparation
- Tags

**Validation Timing:**
- **On blur:** Show errors for touched fields
- **On submit:** Validate entire form, scroll to first error
