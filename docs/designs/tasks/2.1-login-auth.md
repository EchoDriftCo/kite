### Specs

**Container:**
- **Background:** `$background-secondary` (#1e293b) — full-page background
- **Vertical alignment:** Center (flexbox center vertical + horizontal)
- **Padding:** 16px (mobile), 32px (desktop)

**Logo/Title:**
- **Font:** 500 32px/40px (mobile), 500 40px/48px (desktop)
- **Color:** `$text-primary` (#f1f5f9)
- **Letter-spacing:** -0.02em (tighter for logo feel)
- **Margin-bottom:** 32px (mobile), 48px (desktop)
- **Optional:** Replace text with logo image (SVG, max-width 200px)

**Login Card:**
- **Max-width:** 400px
- **Background:** `$surface-default` (#334155)
- **Border-radius:** 16px
- **Shadow:** `$shadow-3`
- **Padding:** 24px (mobile), 32px (desktop)
- **Margin:** 0 auto (center-aligned)

**Card Title ("Sign In"):**
- **Font:** 500 24px/32px
- **Color:** `$text-primary`
- **Margin-bottom:** 24px

**Form Fields:**
- **Email input:**
  - Type: `email`
  - Autocomplete: `email`
  - Placeholder: "you@example.com"
  - Required: true
  - Validation: Email format
- **Password input:**
  - Type: `password`
  - Autocomplete: `current-password`
  - Placeholder: "Enter your password"
  - Required: true
  - Min-length: 6
  - Show/hide password toggle (eye icon, right side)
- **Field margin-bottom:** 16px
- **Field styling:** See `02-components.md` form field specs

**Forgot Password Link:**
- **Font:** 400 14px/20px
- **Color:** `$accent-teal`
- **Alignment:** Right
- **Margin-bottom:** 16px
- **Hover:** Underline, color → `$accent-teal-hover`
- **Action:** Navigate to `/reset-password`

**Sign In Button:**
- **Style:** Primary button (filled, amber background)
- **Width:** 100%
- **Height:** 48px (touch-friendly)
- **Text:** "SIGN IN" (uppercase, 14px, 500 weight)
- **Icon:** None
- **Margin-bottom:** 16px

**Divider ("or"):**
- **Line color:** `$divider` (rgba(241, 245, 249, 0.12))
- **Line height:** 1px
- **Text:** "or" (400 12px, `$text-hint`, centered between lines)
- **Margin:** 16px 0

**Magic Link Button:**
- **Style:** Secondary button (outlined, teal border)
- **Width:** 100%
- **Height:** 48px
- **Icon:** Email icon (left side, 20px, 8px margin-right)
- **Text:** "Sign in with magic link"
- **Margin-bottom:** 24px
- **Action:** Open magic link dialog (email input, send link button)

**Sign Up Prompt:**
- **Font:** 400 14px/20px
- **Color:** `$text-secondary`
- **Alignment:** Center
- **Text:** "Don't have an account? [Sign up]"
- **"Sign up" link:**
  - Color: `$accent-teal`
  - Hover: Underline
  - Action: Open sign-up dialog

### States

**Loading (Sign In clicked):**
- Disable all form fields
- Disable buttons
- Show spinner inside "Sign In" button
- Button text: "Signing in..." (lowercase)

**Error (Invalid credentials):**
- Show error message below "Sign In" button
- **Font:** 400 12px/16px
- **Color:** `$error` (#ef4444)
- **Icon:** `error` icon (16px, left of text)
- **Text examples:**
  - "Invalid email or password"
  - "Account not found"
  - "Too many login attempts. Try again in 5 minutes."
- **Margin-top:** 8px
- **Animation:** Fade in (150ms)

**Success (Logged in):**
- Redirect to `/recipes` (dashboard)
- No intermediate success screen

**Focus States:**
- All inputs: 2px solid `$accent-teal` border
- All buttons: 2px solid `$accent-teal` outline, 2px offset

### Accessibility

- **Focus order:** Email → Password → Forgot Password → Sign In → Magic Link → Sign Up
- **Screen reader labels:**
  - Page title: "Sign In to RecipeVault"
  - Email field: "Email address"
  - Password field: "Password"
  - Show/hide password toggle: "Show password" / "Hide password"
- **Error announcements:** aria-live="assertive" on error message
- **Keyboard shortcuts:**
  - Enter in any field → Submit form (trigger Sign In)

### Responsive Behavior

**Mobile (< 600px):**
- Card padding: 24px
- Logo font-size: 32px
- Full-width card (minus 16px side padding)

**Tablet (600-959px):**
- Card max-width: 400px, centered
- Logo font-size: 36px

**Desktop (960px+):**
- Card max-width: 400px, centered
- Logo font-size: 40px
- Card padding: 32px
