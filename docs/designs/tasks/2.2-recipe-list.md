## Overview

The recipe list is the main dashboard after login. Users view their recipes, browse public recipes, search, filter by tags/equipment/favorites, and create new recipes. This is the most-used page in the app — it needs to be fast, clean, and intuitive.

**Features:**
- View toggle: "My Recipes" vs "Public Recipes"
- Search bar (title search)
- Tag filter (collapsible, multi-select)
- Equipment filter (My Recipes only)
- Favorites filter toggle
- Recipe grid (responsive: 1/2/3 columns)
- AI Generate button
- New Recipe button
- Empty states (different for mine vs public)
- Loading state (skeleton loaders)
- Error state (with retry)

### HTML Structure

```html
<div class="recipe-grid">
  <mat-card *ngFor="let recipe of recipes" 
            class="recipe-card"
            (click)="viewRecipe(recipe.id)"
            tabindex="0"
            [attr.aria-label]="'View recipe: ' + recipe.title">
    
    <button mat-icon-button 
            class="favorite-btn"
            (click)="toggleFavorite(recipe, $event)"
            [attr.aria-label]="recipe.isFavorite ? 'Remove from favorites' : 'Add to favorites'">
      <mat-icon>{{ recipe.isFavorite ? 'favorite' : 'favorite_border' }}</mat-icon>
    </button>
    
    <img *ngIf="recipe.imageUrl" 
         [src]="recipe.imageUrl" 
         [alt]="recipe.title"
         class="recipe-image" />
    <div *ngIf="!recipe.imageUrl" class="recipe-image-placeholder">
      <mat-icon>restaurant</mat-icon>
    </div>
    
    <mat-card-header>
      <mat-card-title>{{ recipe.title }}</mat-card-title>
      <mat-card-subtitle>
        {{ recipe.totalTimeMinutes }} min · {{ recipe.yield }} servings
      </mat-card-subtitle>
    </mat-card-header>
    
    <mat-card-content *ngIf="recipe.tags.length > 0">
      <div class="tags">
        <span *ngFor="let tag of recipe.tags.slice(0, maxVisibleTags)" class="tag-chip">
          {{ tag.name }}
        </span>
        <span *ngIf="recipe.tags.length > maxVisibleTags" class="tag-overflow">
          +{{ recipe.tags.length - maxVisibleTags }} more
        </span>
      </div>
    </mat-card-content>
  </mat-card>
</div>
```

## Accessibility

- **Page title:** "Recipes — RecipeVault"
- **Main landmark:** `<main role="main">`
- **Search label:** "Search recipes" (visible label + aria-label)
- **View toggle:** aria-label on each option ("View my recipes", "View public recipes")
- **Favorites toggle:** aria-label ("Show favorites only" / "Show all recipes")
- **Recipe cards:** 
  - Focusable (tabindex="0")
  - aria-label: "View recipe: [Title]"
  - Enter key navigates to detail
- **Favorite button:** 
  - Separate focus from card
  - aria-label: "Add to favorites" / "Remove from favorites"
- **Tag filter chips:** aria-pressed="true" / "false" (selected state)
- **Empty state:** aria-live="polite" (announces when recipes change)
- **Loading state:** aria-busy="true" on container, aria-live="polite" announces "Loading recipes"
