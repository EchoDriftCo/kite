## 3.3 Discover

### 3.3.1 Discover Page (`/discover`)

**Layout:** Same as recipe list page, but:
- Title: "Discover Public Recipes"
- No "My Recipes" toggle (always public recipes)
- No favorites filter (or show favorited public recipes)
- Search bar + filters work the same
- Recipe cards show "By [Author Name]" attribution (400 12px, `$text-hint`, below title)
- Fork button on each card (or in overflow menu)

### 3.3.2 Fork Recipe Flow

**Trigger:** Click "Fork" button on public recipe card or detail page

**Flow:**
1. Open confirmation dialog:
   - **Title:** "Fork Recipe?"
   - **Message:** "This will create a copy of \"[Recipe Title]\" in your recipe collection. You can edit it freely without affecting the original."
   - **Actions:** "Cancel", "Fork Recipe"
2. If confirmed:
   - Create copy of recipe (all ingredients, instructions, tags)
   - Set `forkedFromRecipeId` to original recipe ID
   - Navigate to new recipe detail
   - Show success snackbar: "Recipe forked successfully! You can now edit your copy."

**Fork Attribution:**
- Show on forked recipe detail (top, below title):
  - **Text:** "Forked from [Original Recipe Title] by [Original Author]"
  - **Font:** 400 14px, `$text-hint`
  - **Link:** Original recipe title links to original (if still public)

## Responsive Behavior

All pages follow same responsive patterns:
- **Mobile:** Single column, full-width cards, stacked buttons
- **Desktop:** Grid layouts (2-3 columns), inline buttons

## Accessibility

- **Circles/Collections lists:** Keyboard navigation, Enter to view
- **Invite dialog:** Email inputs accessible, link copy button clear label
- **Share/Add to dialogs:** Radio/checkbox lists keyboard accessible
- **Fork button:** Clear aria-label "Fork recipe [title]"
- **Screen reader:** Announce member counts, recipe counts, share/fork actions
